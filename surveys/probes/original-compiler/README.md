# Original Compiler Probes

Small Action! programs for comparing `actionc` output against the original
compiler.

## North Star

The current goal is broad original-compiler compatibility for valid programs,
without intentionally reproducing clear original compiler bugs. Prefer the
original Action! code shape, ABI, storage layout, and load-file structure when
probes or the official manual give us evidence. When the original compiler
rejects valid-looking source or emits clearly broken code, keep `actionc`
semantically sane and document the divergence.

Keep compatibility work evidence-driven and narrowly scoped. Probe before
generalizing a pattern, especially for peepholes and register-liveness hints.
At the same time, keep codegen flexible: compatibility choices should live in
recognizable helpers or policy-shaped seams so a later pass can add improved
modern code generation after the compatibility baseline is broad and stable.

Layout:

- `*.act`: probe source files.
- `outputs/actionc/`: output generated by this Rust compiler.
- `outputs/original/`: output captured from the original Action! compiler.

Suggested capture workflow:

```sh
cargo run --bin actionc-emit -- --emit-code surveys/probes/original-compiler/abi_calls.act \
  > surveys/probes/original-compiler/outputs/actionc/abi_calls.hex

cargo run --bin actionc-emit -- --emit-listing surveys/probes/original-compiler/abi_calls.act \
  > surveys/probes/original-compiler/outputs/actionc/abi_calls.lst

cargo run --bin actionc-emit -- --emit-load surveys/probes/original-compiler/abi_calls.act \
  > surveys/probes/original-compiler/outputs/actionc/abi_calls.xex
```

Current compatibility sweep:

```sh
surveys/probes/original-compiler/sweep.sh
```

The sweep regenerates `outputs/actionc/*.com` load files for every VM-captured
probe and compares them byte-for-byte with `outputs/vm/*.COM`. Use
`--update-artifacts` to also refresh the `.hex` and `.lst` files. The allowlist
currently contains the intentional/sane divergences documented in
`VM_PROBE_RESULTS.md`.

The same sweep is also available as an ignored Cargo integration test:

```sh
cargo test --test compatibility -- --ignored
```

Run it before broad codegen refactors or compatibility-sensitive changes.
See `docs/PROBE_SWEEP_PROCESS.md` for the process note on when to run it and
how to interpret results.

`actionc` defaults generated-code listings and label patching to origin `$3000`.
Use `--origin <addr>` to compare against original compiler output captured at a
different load address:

```sh
cargo run --bin actionc-emit -- --emit-listing --origin '$4000' surveys/probes/original-compiler/abi_calls.act
```

The probe sources also begin with:

```action
SET $491=$3000
SET $E=$3000
```

For the original compiler, `$0491` is `codebase` and `$000E`/`$E` is `qcode`, the
current code emission pointer. Setting both keeps emitted code and saved-object
headers anchored at the same origin. `SET *=$3000` does not set the origin; it
writes `$3000` to the current code pointer location.

When capturing original compiler output, save whatever is available:

- raw code bytes as `outputs/original/<probe>.hex`
- listing/disassembly as `outputs/original/<probe>.lst`
- Atari binary load files as `outputs/original/<probe>.xex`
- notes about origin, runtime/library setup, or compiler options as
  `outputs/original/<probe>.notes.md`

Keep each probe narrow. Once behavior is understood and stable, promote the
case into normal tests or a compatibility corpus.

Compatibility note: original Action! writes compiled output directly from live
memory, so storage bytes in saved load files can contain whatever values were
already present in those addresses. `actionc` intentionally emits deterministic
zero-filled bytes for global and routine storage. Compare code layout, labels,
and instructions around those bytes rather than expecting storage contents to
match byte-for-byte.

See `ABI.md` for the consolidated calling convention and storage-layout notes
derived from the current probes.

See `PEEPHOLE_OPTIMIZER.md` for the current understanding of original Action!
local optimization patterns and compatibility guidance for codegen work.
