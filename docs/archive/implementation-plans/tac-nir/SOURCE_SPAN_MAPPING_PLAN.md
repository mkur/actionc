# Source-Span Mapping Plan

Goal: let `actionc-compare` explain generated bytes in terms of source
constructs: routine, statement, expression, declaration, machine block, storage
initializer, and related compiler decisions.

## 1. Add Map Types

Extend `CodegenMap` with source-backed ranges:

```rust
pub source_ranges: Vec<CodegenSourceRange>
```

Initial shape:

```rust
pub struct CodegenSourceRange {
    pub kind: CodegenSourceRangeKind,
    pub name: Option<String>,
    pub source_span: Span,
    pub start: u16,
    pub end: u16,
}

pub enum CodegenSourceRangeKind {
    Routine,
    Statement,
    Expression,
    Declaration,
    StorageInitializer,
    MachineBlock,
}
```

Use exclusive `end`, matching routine ranges.

## 2. Start With Coarse Ranges

Record full routine ranges and top-level statement ranges first. Do not begin
with expression mapping; statement-level mapping should already explain most
first-diff locations.

Also record machine-block ranges early because those can contain inline bytes
that confuse linear disassembly.

## 3. Add Generator Helpers

Add small helpers around `current_absolute_address()`:

```rust
fn record_source_range(...)
fn with_source_range(kind, span, name, f)
```

Keep instrumentation local to `Generator` and avoid changing emitted bytes.

## 4. Instrument Statements

Wrap `generate_stmt` dispatch so each non-empty statement gets a source range.
Skip unsupported or empty ranges initially.

## 5. Instrument Declarations And Storage

Add source ranges for generated storage bytes and skipped ranges where
declaration spans are available. Compatible-mode local storage emitted in
routine preambles should be labelled after statement ranges are working.

## 6. Use The Map In `actionc-compare`

Teach routine diff output to include the nearest source range for the first
instruction diff:

```text
Main len 507 -> 505 (-2)
first diff:
  compat $3268: LDA #$FF
  modern $3268: DEY
  source: pointers.act:42..47 statement assignment
```

The first version can print raw byte spans. Line/column/snippet rendering can
come next.

## 7. Add Line/Column Rendering

Keep source text formatting in tooling, not codegen. `Span` stores byte
offsets; `actionc-compare` can map those to line/column and a one-line excerpt.

## 8. Add Expression-Level Mapping Later

After statement mapping is stable, add expression ranges for high-value areas:

- RHS materialization
- call argument staging
- compare condition generation
- array/pointer address preparation

## 9. Tests

Add tests that assert:

- the map has source ranges for a simple routine
- ranges use source AST spans
- generated ranges point inside `output.bytes`
- compat and modern both produce ranges
- map generation does not affect emitted bytes

## 10. Compatibility Rule

Source-span mapping is observability only. It must not affect generated bytes or
the load-file format.
